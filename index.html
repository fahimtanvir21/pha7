Q&A

1. What does JavaScript work?
First of all, it started with the Parser. As we know a script tag with a source is seen by the HTML parser. Based on the tokens it gets, the parser constructs nodes. It builds an AST (Abstract Syntax Tree) with these nodes. The interpreter comes next, and it traverses over the AST and generates byte code. Line by line, it reads the code. The AST is removed after the byte code has been created, freeing up memory. The Profiler then monitors and watches code in order to optimize it, and the compiler then works ahead of time to make a translation of the code that has been created and compiles it down to a machine readable language. It then optimized the code to make it run faster. Thus, JavaScript works.

2. What does JavaScript Event Loop do?
The event loop is the key of asynchronous programming in JavaScript. Although, JavaScript executes all operations on a single thread, it offers the appearance of multi threading through the use of a few data structures.The call stack is in charge of keeping track of all the actions in the queue that need to be completed. When a function completes, it is removed from the stack. After that an Async function is submitted to a browser API whenever it is called. These are built-in APIs in the browser. The API initiates its own single threaded activity based on the command that received from the call stack.
